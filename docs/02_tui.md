# Как мы создали TUI для Kubernetes на Go (yak8sui)

В этой статье суммирован процесс создания легковесного терминального интерфейса (TUI) для просмотра подов Kubernetes с использованием языка Go и библиотеки `tview`. Этот документ можно использовать в качестве архитектурной справки для проекта.

---

## Архитектурный подход

При проектировании приложения мы разделили зоны ответственности на два независимых слоя:
1. **Data Layer (Пакет `k8s`)**: Чистая логика взаимодействия с Kubernetes API через официальный `client-go`. Он ничего не знает о том, как данные будут отображаться.
2. **Presentation Layer (Файл `main.go`)**: Интерфейс пользователя, построенный на базе `tview`. Он отвечает исключительно за визуализацию, обработку ввода и обновление экранов.

Такое разделение позволяет легко тестировать бизнес-логику отдельно от UI и в будущем безболезненно переносить компоненты.

---

## Почему был выбран `tview`, а не `Bubble Tea`?

Для разработчика, который только начинает погружаться в экосистему Go, порог входа имеет решающее значение:
* **Bubble Tea (Charmbracelet)** использует реактивную архитектуру Elm (Model-View-Update). Она концептуально прекрасна, но требует управления сложными потоками сообщений (`Msg`) и команд (`Cmd`), что перегружает код на начальном этапе.
* **tview** предоставляет классический императивный ООП-подобный подход (как конструктор LEGO). Мы создаем готовые виджеты (таблицы, списки, сетки), настраиваем их свойства «сверху вниз» и монтируем в приложение. Это делает код линейным, понятным и простым для отладки.

## Пошаговый процесс реализации

### Шаг 1: Сбор данных из кластера (`pkg/k8s/k8s.go`)
Мы инкапсулировали работу с API в функцию `GetPodNames(namespace string)`. Она выполняет следующие действия:
* Находит домашнюю директорию пользователя для локализации `.kube/config`.
* Строит конфигурацию подключения с помощью `clientcmd.BuildConfigFromFlags`.
* Создает `clientset` и выполняет запрос к CoreV1 API для получения списка подов в заданном namespace.
* Маппит результат в простую и понятную структуру `PodInfo { Name, Status }`.

### Шаг 2: Создание каркаса интерфейса (`cmd/yak8sui/main.go`)
В точке входа приложения мы инициализировали базовые компоненты `tview`:
* **`tview.NewTable()`**: Интерактивный компонент таблицы. Мы настроили отображение рамок (`SetBorders(true)`) и включили навигацию по строкам (`SetSelectable(true, false)`).
* **`tview.NewTextView()`**: Строка подвала (footer), служащая панелью подсказок для пользователя.
* **`tview.NewGrid()`**: Сетка, которая делит экран по вертикали на две зоны: основная таблица (занимает всё доступное пространство `0`) и подвал фиксированной высоты в 1 строку (`1`).

### Шаг 3: Динамическое обновление данных
Вместо статичной отрисовки мы обернули логику заполнения таблицы в функцию-замыкание `refreshData()`. При её вызове:
1. Старые строки таблицы (кроме шапки) очищаются с помощью метода `table.RemoveRow()`.
2. Из пакета `k8s` запрашивается свежий срез подов.
3. В цикле генерируются новые ячейки (`tview.NewTableCell()`).
4. Применяется **цветовая индикация статусов** через библиотеку `tcell`: если статус пода равен `Running`, он подсвечивается зеленым (`tcell.ColorGreen`), в противном случае — красным.

### Шаг 4: Перехват пользовательского ввода
С помощью метода `table.SetInputCapture()` мы определили реакцию на клавиатурные события:
* Нажатие `Esc` или `Ctrl+C` корректно завершает TUI-сессию через `app.Stop()`.
* Нажатие английской клавиши `r` или `R` повторно триггерит функцию `refreshData()`, реализуя механизм ручного обновления данных без перезапуска утилиты (поведение полностью аналогично оригинальному `k9s`).

## Подводные камни и их решение

### 1. Ошибка `terminal entry not found: term not set` в IDE
При запуске TUI из встроенной консоли GoLand/VSCode приложение падало.
* **Причина**: Консоль отладки IDE по умолчанию работает в режиме "сырого" потока ввода-вывода и не передает тип эмулируемого терминала.
* **Решение**: В конфигурации запуска IDE была явно прописана переменная окружения `TERM=xterm-256color` и включена опция **"Emulate terminal in output console"**.

### 2. Исчезновение символа `[r]` в подвале
При включенных динамических цветах (`SetDynamicColors(true)`) текст `[r] Обновить данные` превращался в `Обновить данные` зеленого цвета.
* **Причина**: Парсер строк `tview` воспринимал скобки `[r]` как внутренний тег цвета (r = red).
* **Решение**: Для панели подсказок парсер цветов был отключен (`SetDynamicColors(false)`), что позволило безопасно выводить любые комбинации квадратных скобок как обычный текст.
